# Source Document: WPLDS 2015 Handbook
**Context:** West Point Leader Development System (WPLDS)
**Relevance to SAX:** System architecture, continuous assessment, and documentation standards

## Core Thesis
The West Point Leader Development System (WPLDS) is not a training program but a living, adaptive system designed to produce a specific human outcome through 47 months of integrated, progressive, multi-domain experience. The operating principle is that "every interaction is a developmental opportunity" [1].

## Key Mechanisms

### The Three Dimensions
The structure is built on three dimensions operating simultaneously:
1. **Individual Leader Development:** Building the person's character and competence.
2. **Leadership Development:** Building competence in social and organizational contexts.
3. **Culture of Character Growth:** The environment that makes the first two possible [1].

### The Feedback Architecture
The system does not wait for annual reviews. It assesses continuously, documents everything, and uses those assessments to adapt the development path:
* **Cadet Observation Reports:** Captured in the moment.
* **Periodic Development Reviews:** Conducted four times per semester using 360-degree evaluation.
* **Cadet Development Reports:** Formalized once per semester [1].

## Application to The Exchange Protocol
WPLDS solved the same problem SAX is solving: how to build a system that develops people consistently, at scale, across multiple domains, with accountability at every level, without depending on any single person to run it.

The WPLDS architecture maps directly to the SAX model:
* **The 47-Month Arc** becomes the **4-Year Athletic Career Arc**.
* **The Three Dimensions** become the **Three-Tier Ecosystem** (Self, Landscape, Organization).
* **The Cadet Observation Report** becomes the **Execution Trace Unit (ETU)**.
* **The Periodic Development Review** becomes the **Map Verification Cycle**.
* **The Cadet Development Report** becomes the **Work Ledger (Portfolio)**.

The standard WPLDS set—documentation is mandatory, feedback loops are built into the schedule, and every person has a defined role—is the standard the SAX governance architecture must meet.
